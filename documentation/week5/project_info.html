Week 5 progress update

The common method in the beginning of the development of this type of web elements is, first, to make sketches about the interface that we are going to show and a prototype where the functionality of this is shown. To create the first approach to the interface we decided to list what elements we believe necessary, we obtained the following:

  • Preview space: area where we can see graphically the situation and the final result of the animation.
  • Timeline: here we show where the keyframes we have created are located, with respect to time.
  • Keyframe list: list with the positions, rotation, scale that we want to interpolate in each keyframe.
  • Selector: list with each object of the scene.
  • Code display: shows the code of the animation made.

With these elements we can start to create the basic outline of the web page. For this, we have used draw.io

Once we have made the first scheme, we place placeholders (non-interactive images) to obtain a first image of the design and its elements. Starting from that first image, we can build the base of the interactive interface. In this case, we start with the preview zone, which occupies most of the page and where the animations are transformed. For this we create a special zone that delimits the movement of the objects of the scene although, in the first prototype, they can not be moved. To this we will add an object selector for the scene, if you click on one of the objects in the preview, for the prototype we have added two objects. One of the most important parts is the timeline, we have built in the preview space and it works as a player (play / pause) in which you can move the time in which you are. Along with the timeline we have implemented the list of keyframes, since they will work together, we have added 4 frames as an example and, in a more developed stage, to select and edit the keyframes placed on the timeline and the transformations of the objects at that moment.
